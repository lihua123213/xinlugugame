# 像素农场 — 完整项目技术文档

## 一、项目概述

**像素农场**是一款基于 HTML5 Canvas 的像素风农场经营游戏，包含完整的农业系统、僵尸地牢战斗、NPC 交互、局域网联机等玩法。全部代码为单文件架构（`farm-game.html` 75KB），无任何外部图片资源，所有视觉元素均由 Canvas API 逐像素绘制。

---

## 二、技术栈

| 技术 | 用途 |
|------|------|
| **HTML5 Canvas** | 全部游戏画面渲染，960×640 分辨率，`image-rendering: pixelated` 确保像素锐利 |
| **原生 JavaScript (ES6)** | 游戏逻辑，使用 class 语法组织模块 |
| **Node.js + ws 库** | WebSocket 服务器，处理局域网联机通信 |
| **localStorage** | 存档/读档系统 |

## 四、核心系统详解

### 4.1 地图系统

**地图尺寸**：80×35 格（2560×1120 像素），每格 32×32px。

**瓦片类型**：
```
T.GRASS=0, T.WATER=1, T.STONE=2, T.WOOD=3, T.PATH=4, T.SAND=5, T.BRIDGE=6
```

**生成算法**：
- 河流：正弦函数蜿蜒 `y0 = 8 + sin(x*0.2)*3 + sin(x*0.45)*1.5`
- 第二条河：`y0 = 30 + sin(x*0.18)*4 + cos(x*0.3)*2`
- 山脉：y<5 区域 60% 概率生成石头
- 森林：右半区 40% 概率生成松树/橡树/果树
- 道路：Bresenham 直线算法连接各建筑
- 花朵：水边/路边额外 20% 概率生成装饰花

**建筑**：使用 2×2 瓦片标记（`shop1/shop2`、`house1/house2`、`rest1/rest2`、`smithy1/smithy2`、`p2room1/p2room2`、`mine`），渲染函数统一处理。

### 4.2 渲染引擎

**分层渲染**（每帧 60fps）：
1. `_rWorld` — 地形瓦片（逐格绘制，可见区域裁剪）
2. 作物层 — CropMgr 绘制生长状态
3. NPC/鸡/僵尸 — 实体渲染
4. 玩家角色 — 16×22 像素精灵，4 方向动画
5. `_rLight` — 平滑半透明光照叠加（非 dither，避免方块感）
6. 天气粒子 — 雨/雪/落叶
7. `_rHUD` — 右上角信息面板 + 底部工具栏
8. 状态面板 — 背包/商店/饭店/铁匠铺/对话框

**像素风技巧**：
- 所有 `fillRect` 坐标为整数，避免抗锯齿
- 树木用 `arc()` 绘制圆形树冠，加亮色高光圆
- 水面用 4 色横条 + 正弦波光动画
- 光照用 `rgba` 半透明叠加，黄昏加暖色滤镜
- 角色阴影用 `ellipse()` 椭圆

### 4.3 玩家系统

**碰撞检测**：4 角点法，每个角检查 `isSolid(map[ty][tx])`：
```javascript
_col(nx, ny, map) {
  for each corner [nx±hw, ny±hh]:
    tx = floor(px/TS), ty = floor(py/TS)
    if isSolid(map[ty][tx]) → blocked
}
```

**动画**：4 帧行走循环，`Math.sin(frame*PI/2)` 弹跳效果。

### 4.4 农业系统

**作物状态机**：
```
seed → stage0(嫩芽) → stage1 → stage2 → mature(可收获)
每阶段需要 N 天浇水，不浇水不生长
```

**季节枯萎**：每季 2 天，季节变更时检查作物种子是否匹配当前季节，不匹配则枯萎。

### 4.5 僵尸地牢系统

**进入方式**：夜晚（h≥18 或 h<6）走到桥边按空格 → 确认进入

**地牢地图**：30×20 格独立地图，石头墙壁 + 散落水池 + 绿色传送门出口

**僵尸 AI 状态机**：
```
巡逻(随机走动) → 检测玩家(距离<200) → 追击(速度0.7) → 攻击(距离<40, 冷却45帧)
```

**战斗系统**：
- 空格键触发攻击，面向方向生成 28×28 碰撞框
- 与僵尸 hitbox(16×18) 做 AABB 碰撞检测
- 僵尸 HP=3+wave，每击-1，击杀后 35% 掉粉色宝石
- 铁剑效果：每击额外扣 1 血（等效双倍伤害）
- 铁甲效果：僵尸伤害从 5 降到 3

**胜利条件**：拿到粉色宝石 → 回家 → 触发胜利画面

### 4.6 联机系统

**架构**：P1 权威模式（Host 运行全部逻辑）

**通信协议**：
- P1 → 服务器 → P2：JSON `{type:'state', p1:{x,y,d,m,a,t}, p2:{x,y,d,m,a,ac}, t:{h,m,d,s}, g:gold}`
- P2 → 服务器 → P1：JSON `{type:'p2_input', keys:{KeyW:true, ...}}`
- 地图：HTTP POST `/api/map` 一次性缓存，P2 通过 GET 获取

**服务器（server.js）**：
```javascript
// HTTP: 静态文件 + /api/map 缓存端点
// WebSocket: host/client 角色分配 + 消息转发
// 关键修复: data.toString() 确保浏览器收到字符串而非 Buffer
```

**P2 渲染**：`client.html` 独立渲染器，接收状态后用相同瓦片绘制逻辑渲染画面。

### 4.7 NPC 系统

3 个 NPC（老李、小梅、铁匠王），每个有：
- 好感度 0-10（送礼增加）
- 喜好列表（老李喜欢花/防风草，小梅喜欢花/番茄，铁匠王喜欢矿石/石头）
- 对话随好感度变化（5 档对话）

### 4.8 饭店系统

3 种食物：套餐(50G/40体力)、热汤(30G/25体力)、面包(15G/12体力)
点击按钮购买，同时恢复体力和 HP。

### 4.9 铁匠铺系统

- 铁甲：3 铁矿石 → 僵尸伤害减半
- 铁剑：2 铁矿石 → 攻击伤害+50%
- 铁矿石来源：矿洞挖掘（35% 概率掉落）

## 五、关键问题与解决方案

### 问题 1：动作系统失效
**现象**：锄地/砍树等操作不生效
**根因**：`_action()` 方法中 `useTool()` 有动画阻塞检查（`this.ut` 为 true 时禁止新操作），但动画时长 22 帧 > 冷却 20 帧，导致交替丢帧
**修复**：移除 `useTool()` 中的动画阻塞，改为直接执行 + 设置动画

### 问题 2：Tab 背包闪退
**现象**：按 Tab 打开背包瞬间关闭
**根因**：`window.addEventListener('keydown')` 和 `_upd()` 中都有 Tab 处理，两个处理器同时触发
**修复**：移除 window 级别的 Tab 处理，统一由 `_upd` 中的 `edge('Tab')` 管理

### 问题 3：矿洞无法操作
**现象**：进入矿洞后不能移动、不能挖矿
**根因**：矿洞使用屏幕坐标（CW/2, CH/2），岩石生成在随机位置，但玩家固定在中心无法移动
**修复**：矿洞改为世界坐标系，玩家可 WASD 移动，岩石生成在 4-24 格范围

### 问题 4：游戏逐渐卡顿
**现象**：长时间游玩后帧率下降
**根因**：树桩对象 `_stumps` 只增不删，`for...in` + `delete` 行为不一致
**修复**：
- 用 `Object.keys()` 代替 `for...in` 遍历
- 每 1200 帧自动清理过期树桩
- 睡觉时清空全部树桩
- 粒子数硬上限（动作 200 + 天气 80）

### 问题 5：回家直接跳到第二天
**现象**：按空格回家直接变成第二天早上
**根因**：`_sleep()` 直接调用 `advanceDay()`，没有中间状态
**修复**：白天回家 → 设置 tick=21600（对应 h=18），变为夜晚；夜晚回家 → 正常睡觉到第二天

### 问题 6：光照方块感
**现象**：日夜变化时出现明显的方块阴影
**根因**：光照使用棋盘格 dither（`((dx>>2)+(dy>>2))&1`），4×4 像素块太明显
**修复**：改为单次 `ctx.fillRect(0,0,CW,CH)` 半透明叠加，平滑过渡

### 问题 7：联机 P1 卡死
**现象**：P2 连接后 P1 游戏循环停止
**根因**：`_sendState()` 每帧执行 `JSON.stringify(map)` — 80×35 地图的 JSON 约 20KB，阻塞主线程
**修复**：
- 地图通过 HTTP POST 一次性发送到服务器缓存
- 每帧只发送紧凑的状态数据（约 200 字节）
- P2 通过 GET `/api/map` 获取地图

### 问题 8：P2 收不到状态
**现象**：P2 有地图但画面不更新
**根因**：服务器用 `ws.send(data)` 转发 Buffer，浏览器收到的是 Blob 不是 string
**修复**：`client.send(data.toString())` 确保发送字符串

### 问题 9：P2 WebSocket 反复断开
**现象**：P2 连接后立即断开，反复重连
**根因**：服务器连接时发送巨大 map JSON 导致断开
**修复**：移除 WS 发送 map（改用 HTTP），只通过 WS 发送轻量状态

### 问题 10：P2 显示等待但收不到状态
**现象**：P2 有地图、WS 连接正常，但 `joined` 始终 false
**根因**：`_sendState` 发送 JSON `{type:'state',...}`，但客户端只检查 `S|...` 紧凑格式
**修复**：客户端 `onmessage` 同时处理 JSON 和紧凑格式

### 问题 11：_p2Input 未初始化
**现象**：P2 连接后 `_updP2` 报错 `Cannot read properties of undefined`
**根因**：构造函数中缺少 `this._p2Input={}` 初始化
**修复**：在 Game 构造函数中补全所有联机相关字段初始化

## 六、性能优化

| 优化项 | 效果 |
|--------|------|
| Canvas context 缓存 (`this._ctx`) | 避免每帧 getContext 调用 |
| 可见区域裁剪（只渲染屏幕内瓦片） | 80×35 地图只渲染 ~30×20 |
| 树桩自动清理（Object.keys + 定时 delete） | 避免内存泄漏 |
| 粒子数硬上限 | 防止 GC 压力 |
| 联机状态紧凑格式（200字节/帧） | 避免 JSON.stringify 阻塞 |
| 地图 HTTP 传输（非 WS） | 避免大消息导致连接断开 |
